# 数据获取流程

## 场景 A：用户已登录 + 给 URL

最快路径：playwright-cli attach 用户已经开的 Edge。

### 步骤

1. **重启 Edge 启用远程调试**（一次性）
   ```bash
   taskkill //F //IM msedge.exe
   "<EDGE_PATH>" --remote-debugging-port=9222 &
   sleep 4
   curl -s --max-time 3 http://127.0.0.1:9222/json/version  # 确认调试端口活
   ```

   `<EDGE_PATH>` = Edge 可执行文件位置。Windows 上通常是：
   ```
   "C:/Program Files (x86)/Microsoft/Edge/Application/msedge.exe"
   ```
   也可能装在 `C:/Program Files/Microsoft/Edge/Application/msedge.exe`（64 位）。
   用 `where msedge` 或文件管理器搜 "msedge.exe" 找实际位置。

   Edge 恢复 tab + 登录态是默认行为（现代 Edge 行为）。

2. **attach playwright-cli**
   ```bash
   playwright-cli attach --cdp=http://127.0.0.1:9222
   ```
   attach 上后会列出现有 tabs，找到 homework 详情那个（标题"作业详情"），
   用 `playwright-cli tab-select <N>` 切过去。

3. **切到"答题解析"标签**
   顶部有 `作业结果 / 答题解析 / 作业评语` 三个 tab，点中间那个。

4. **逐 section 抓数据**
   - 顶部有 `阅读M1 / 阅读M2 / 听力M1 / 听力M2` 四个按钮，按 section 切。
   - 每个 section 都有题目列表（1, 2, 3...），
     用 snapshot 拿到每题的 ref（`eNNNN` 格式）。
   - 对每道错题：
     - click 题号 → snapshot → 提取题干、4 选项、学生/正确答案
     - 错题才需要详细 snapshot，对的题不用
     - 阅读题：snapshot 里直接有"阅读材料"区（文字或图）
     - 听力题：snapshot 里点 "原文" 按钮 → 再 snapshot → tooltip 里有 transcript

5. **阅读材料的图**（生活类阅读常用）
   ```javascript
   Array.from(document.querySelectorAll('img'))
     .map((img, i) => i + ': ' + (img.src || 'none').substring(0, 200))
   ```
   拿到 URL 后 `curl -L` 下载到 study/ 目录，用 Read tool 直接看图。

6. **保存所有 snapshot** 到 `study/wrong/<section>-<Q>.yml`，按需取用。

### 关键坑

- **refs 用一次就失效**：每次 click 后必须重新 snapshot 拿新 ref。
  解决：批量 click + snapshot 之后从 snapshot 文件里 grep ref。
- **"原文" 按钮要用 eval 点**：`eval` + 找 textContent === "原文" 的 button
  比靠 ref 点更稳。
- **section 切换会重置 refs**：切 section 后所有 ref 都变。
  解决：每个 section 切进去后立刻 snapshot 拿一份"当前所有 ref"的 map。

## 场景 B：用户没登录

两种选：
1. 让用户先在浏览器里登录，再走场景 A
2. 让用户把内容复制粘贴成文件（最稳但累）

不要在 skill 里硬塞登录流程（账号密码问题）。

## 场景 C：用户直接给内容

直接进入"写报告"步骤，跳过抓数据。


## 抓完之后的数据结构

心中要有这个模型：

```
exam {
  name: "第五次模考-真题16"  (从页面 h2 标题取)
  date: "2026.06.20"
  url: ...
  sections: {
    reading_m1: { 错题: [1, 3, 4, 12, 13] },
    reading_m2: { 错题: [1, 3, 6] },
    listening_m1: { 错题: [3, 7, 8, 13, 14, 16] },
    listening_m2: { 错题: [1, 2, 14] }
  }
}

question {
  type: "词汇填空" | "生活类阅读" | "学术型阅读" | "听并回应" | "对话" | "讲座" | "公告"
  section: "M1" | "M2"
  num: 13
  score: "9/10" | "0/1"
  text: <原文或图 OCR>  // 词汇填空 = 带空格的段落
  question: <题干>
  options: { A: "...", B: "...", C: "...", D: "..." }
  student: "A"
  correct: "B"
  transcript: <只听力有>  // 完整 transcript
}
```

错题列表出来后，按 `references/SKILL.md` 的"题型 → 处理模板"逐题写。
