# 数据获取流程

## 场景 A：用户已登录 + 给 URL（最常见）

最快路径：playwright-cli attach 用户已经开的浏览器。

### 一次性启动（只第一次跑）

> ⚠️ 杀进程命令会强制结束**所有**该浏览器的进程，未保存的 tab / 下载 / 表单都会丢。

按 OS 走对应分支：

**Windows**：
```bash
taskkill //F //IM msedge.exe 2>/dev/null  # Edge
"C:/Program Files (x86)/Microsoft/Edge/Application/msedge.exe" --remote-debugging-port=9222 >/dev/null 2>&1 &

taskkill //F //IM chrome.exe 2>/dev/null  # Chrome
"C:/Program Files/Google/Chrome/Application/chrome.exe" --remote-debugging-port=9222 >/dev/null 2>&1 &

sleep 4
curl -s --max-time 3 http://127.0.0.1:9222/json/version
```

**macOS**：
```bash
pkill -f "Microsoft Edge" 2>/dev/null; true
"/Applications/Microsoft Edge.app/Contents/MacOS/Microsoft Edge" --remote-debugging-port=9222 &

pkill -f "Google Chrome" 2>/dev/null; true
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --remote-debugging-port=9222 &

sleep 4
curl -s --max-time 3 http://127.0.0.1:9222/json/version
```

**Linux**：
```bash
pkill -f "microsoft-edge" 2>/dev/null; true
microsoft-edge --remote-debugging-port=9222 &

pkill -f "chrome" 2>/dev/null; true
google-chrome --remote-debugging-port=9222 &

sleep 4
curl -s --max-time 3 http://127.0.0.1:9222/json/version
```

启动后写 `~/.msh-skill-state.json` 缓存浏览器名。

### attach playwright-cli

```bash
playwright-cli attach --cdp=http://127.0.0.1:9222
```

输出会列出现有 tabs。**用 eval 找 homework 详情 tab 切过去**，比 tab-select ref 稳：

```bash
playwright-cli --s=default eval "() => {
  const t = Array.from(document.querySelectorAll('a, [role=tab]')).find(el => /homework\/student\/\d+/.test(el.href || el.dataset?.url || ''));
  if (t) { t.click(); return 'clicked: ' + (t.href || ''); }
  // 退化：找所有 tab URL 含 student/
  const all = Array.from(document.querySelectorAll('a[href*=\"/homework/student/\"]'));
  if (all[0]) { all[0].click(); return 'clicked: ' + all[0].href; }
  return 'not found, current url: ' + location.href;
}"
```

attach 后**别再用 tab-select ref**——它跨进程不稳定。


## 流程（快进版）

### Step 1：进"答题解析" + 抓页面"全局状态"

一次性发：

```bash
playwright-cli --s=default click "e答题解析ref"  # 第一次还可用 ref
# 之后一律用 eval 找"答题解析"按钮
playwright-cli --s=default eval "() => {
  const btn = Array.from(document.querySelectorAll('li, button, [role=tab]')).find(e => e.textContent.trim() === '答题解析');
  if (btn) { btn.click(); return 'clicked'; }
  return 'not found';
}"
```

### Step 2：枚举 4 section 的错题（一次 eval）

```bash
playwright-cli --s=default eval "() => {
  // 当前激活 section 的所有题号 + 对错
  const items = Array.from(document.querySelectorAll('div'))
    .filter(d => /^\(\d+\)\s*[✓✗]\$/.test(d.textContent.trim().replace(/\s+/g, '')))
    .filter(d => !d.querySelector('div'))  // 取最里层
    .map(d => {
      const m = d.textContent.match(/\((\d+)\)\s*([✓✗])/);
      return m ? {n: +m[1], wrong: m[2] === '✗'} : null;
    }).filter(Boolean);
  // 去重
  const seen = new Set();
  return items.filter(i => seen.has(i.n) ? false : (seen.add(i.n), true));
}"
```

输出形如 `[{n:1, wrong:true}, {n:8, wrong:true}, {n:17, wrong:true}]`。

存到脑子里 / 写到 `study/wrong/<section>-list.json`。

### Step 3：一次性抓右栏所有错题 innerText

**关键优化**：切完 section 后**别逐题 click**。直接读右栏（题详情区）innerText，自己用 `?` 提取题号/题干/选项/学生/正确。

```bash
playwright-cli --s=default eval "() => {
  // 找到题详情容器（多个候选）
  const candidates = [
    '.question-detail', '[class*=QuestionDetail]', '[class*=questionDetail]',
    '[class*=Detail]', 'main', 'article'
  ];
  for (const sel of candidates) {
    const el = document.querySelector(sel);
    if (el && el.innerText.length > 200) return el.innerText;
  }
  // 退化：取 body 最后 4000 字符
  return document.body.innerText.slice(-4000);
}"
```

如果题详情区**只展示当前选中题**（多数情况如此），还是要 click 切换——但 click 后**立即 dump innerText**，别用 snapshot：

```bash
# 一次性循环点完所有错题 + 收集 innerText
playwright-cli --s=default eval "(wrongNums) => {
  return new Promise(async resolve => {
    const results = {};
    const allItems = Array.from(document.querySelectorAll('div'))
      .filter(d => /^\(\d+\)\s*[✓✗]\$/.test(d.textContent.trim().replace(/\s+/g, '')))
      .filter(d => !d.querySelector('div'));
    for (const n of wrongNums) {
      const item = allItems.find(d => d.textContent.includes('(' + n + ')'));
      if (!item) { results[n] = 'NOT FOUND'; continue; }
      item.click();
      await new Promise(r => setTimeout(r, 400));  // 等待 DOM 更新
      const detail = document.body.innerText;
      // 提取题详情段（从"题目前引导"或"题干"到下一个题号前的部分）
      const start = detail.search(/(题目前引导|题干|题号|得分)/);
      let end = detail.length;
      const nextMatch = detail.slice(start + 50).match(/\n\(\d+\)\s*[✓✗]/);
      if (nextMatch) end = start + 50 + nextMatch.index;
      results[n] = detail.slice(start, end);
    }
    resolve(results);
  });
}" '[1,8,17]'
```

⚠️ **playwright-cli 的 eval 参数传 JSON 字符串**。array 写 `'[1,8,17]'`，str 写 `'"text"'`。函数体里 `(wrongNums) => { ... }` 用 Promise 异步收集。

### Step 4：听力 transcript 一次拿全

听力题要把"原文" popover 全部触发，**再** innerText 拿全。

```bash
playwright-cli --s=default eval "(wrongNums) => {
  return new Promise(async resolve => {
    const transcriptByQ = {};
    const allItems = Array.from(document.querySelectorAll('div'))
      .filter(d => /^\(\d+\)\s*[✓✗]\$/.test(d.textContent.trim().replace(/\s+/g, '')))
      .filter(d => !d.querySelector('div'));
    for (const n of wrongNums) {
      const item = allItems.find(d => d.textContent.includes('(' + n + ')'));
      item.click();
      await new Promise(r => setTimeout(r, 400));
      // 找 "原文" 按钮并 click 触发 popover
      const btn = Array.from(document.querySelectorAll('button, div, span'))
        .find(e => e.textContent.trim() === '原文' && e.children.length === 0);
      if (btn) {
        btn.scrollIntoView({block: 'center'});
        btn.click();
        await new Promise(r => setTimeout(r, 300));
        // 抓 .el-popover 的 innerText
        const pop = Array.from(document.querySelectorAll('.el-popover'))
          .find(p => getComputedStyle(p).display !== 'none' && p.offsetHeight > 0);
        transcriptByQ[n] = pop ? pop.innerText : '(no popover)';
      } else {
        transcriptByQ[n] = '(no 原文 btn)';
      }
    }
    resolve(transcriptByQ);
  });
}" '[5,14,26,27]'
```

### Step 5：拼成内部数据结构

把上面 4 步的输出合到一起（agent 在内存里做），形如：

```
exam {
  name: "【26.6.13-模考-第四次模考：真题12】"
  sections: {
    R-M1: { 错题: [1, 8, 17] },
    R-M2: { 错题: [1] },
    L-M1: { 错题: [5, 14, 26, 27] },
    L-M2: { 错题: [4, 5, 9, 13] }
  }
}

Q { section, num, type, score, text/transcript, question, options{ABCD}, student, correct }
```

**注意 M1-Q8 这种数据 bug**（题列表 ✗ 但实际学生答对）——以右栏"答题结果"段为准。✓ 正确 = 不算错题，跳过。

### Step 6：保存（按需）

存到 `study/wrong/<id>/<section>-<Q>.md` 每题一个文件，方便回查 / 复用。**别等全做完再写文件**——边抓边写，省得最后出错要重来。

### Step 7：写报告 + 收尾

按 `SKILL.md` 风格硬要求，写 `~/Desktop/阅读.txt` 和 `~/Desktop/听力.txt`。

输出**只写两行**：文件路径 + 考试名。不复述做了什么。


## 关键坑（必读）

- **refs 用一次就失效**：每次 click 后必须重新 snapshot 拿新 ref。**改用 eval 找元素**——根本不用 ref。
- **"原文" 按钮在视口外**：先 `scrollIntoView({block:'center'})` 再 click。
- **section 切换会重置 DOM**：切完等 300ms 再 eval，DOM 可能没渲染完。
- **popover 不显示**：`el-popover` 用 click 触发不是 hover。click 后等 300ms 再查 `getComputedStyle(p).display !== 'none'`。
- **Q8 数据 bug**：题列表 ✗ 0/1 但右栏 ✓ 正确。以右栏"答题结果"段为准。
- **同一题组多题**：M1-Q14/Q26/Q27/Q9 共享同一段 talk/对话的 transcript。**用 (n) 题号 click 后再读 popover**，不要在 transcript 缓存里假设共享——因为切换错题时右栏 transcript 也会被覆盖，必须每次重新触发。

## 场景 B：用户没登录

1. 让用户先在浏览器里登录，再走场景 A
2. 让用户把内容复制粘贴成文件（最稳但累）

不要在 skill 里硬塞登录流程。
