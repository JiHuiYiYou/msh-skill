# 模板对照：手写版 vs AI 版 vs 目标版

## 同一道听力题

**原题**：
> Prompt: "What does the woman recommend the man do with his parents?"
> 选项 D 选了一个不沾边的，学生选 D。

### 手写版（参考样本 `手写错题分析/听力1.txt` 7 题）

```
7）
 "What does the woman recommend..."（女士推荐男士做什么）。
在对话的最后，男士提到父母来看他的画展，女士回应说："And you should take your parents to the Italian place on Front St. it's really good."（你应该带父母去Front街的那家意大利餐厅，真的很不错。）
A. Eat at a restaurant（在餐厅吃饭）：完全对应 "Italian place"（意大利餐厅），这是女士给出的明确建议。
B. Visit an art museum（参观艺术博物馆）：这是男士自己要带父母去的地方（"they want to see them"），不是女士推荐的。
C. Bring them to an Italian class（带他们去上意大利语课）：原文是 "Italian place"（餐厅），不是 "Italian class"（课程），属于偷换概念。
D. Listen to a band（听乐队表演）：这是男士原本想去但去不成的音乐节，与带父母的建议无关。
错因：只听到了Italian，后面具体是什么没注意没听到
```

**特征**：
- 题号 `7）` 中文括号
- 错因放最后，一句话口语化
- 选项分析基本按 ABCD 顺序，但手写版不一定全覆盖
- 引用了原文（带引号）
- 没框线没结构


### AI 版（要避免的反例）

```
────────────────────────────────────────────
【听力 M1 - 对话 Q7】
────────────────────────────────────────────

■ 原文（题目组音频）
"Listen to a conversation..."

■ 错因分析
未注意到原文中"Italian place"指的是餐厅而非课程，偷换概念。

■ 选项分析
A. Eat at a restaurant → 对应"Italian place"，正确。✔
B. Visit an art museum → 男士要带父母去的地方，不是女士推荐。✘
C. Bring them to an Italian class → 偷换概念。✘
D. Listen to a band → 与带父母无关。✘

────────────────────────────────────────────
```

**问题**：
- 框线 `─` 标题 `【】` 段落标记 `■` 全用上了
- 错因抽象"未注意到..."
- 选项分析像答题卡，每项一行 + 简评
- 没有任何"思路过程"暴露


### 目标版（这次 skill 写出来的样子）

```
M1
对话

7）原文:
"Listen to a conversation.
 A: ...
 B: ...And you should take your parents to the Italian place on
    Front St. it's really good."

题: What does the woman recommend the man do with his parents?
我选 D (Listen to a band).

错因: 我只听到 Italian, 后面具体是什么没注意. 听到 Italian 就想"和意大利有关",
脑子一抽往课程/语言方向蹦, 没接住"Italian place on Front St"是具体那家餐厅.
正确的 A (Eat at a restaurant) 直接对应"Italian place", 人家都把餐厅名字告诉你了.
```

**和手写版的差异**（要承认）：
- 选项分析只写错的那个 + 正确答案一句，其它不写
- 错因在分析前（手写版放最后）—— agent 选前面更易读

**和 AI 版的差异**：
- 错因"我脑子一抽往课程方向蹦"暴露思路
- 没有框线标题
- 没有"未注意..."那种抽象表述
- 用了 `我选 X` 的口语化开题
- 没有 → ✔ ✘ 这种符号
- 没有"建议加强..."那种结构化尾巴


## 词汇填空的对比

### 手写版

```
M2
1-10 
社群环境让艺术家能够从同行那里获得反馈，尝试新的创作技法，从而丰富他们的艺术创作实践。

错词：
experiment 实验
artistic 艺术的
```

**特征**：
- 短小
- "错词"二字开列
- 不分析为什么错（因为这次是错多个）


### 目标版（针对只错 1 个的填词题）

```
M2
填词

1）9/10. 错了第一个, "A vib____ period" 我没反应过来, 看到 vib 想到 diabolic/symbolic
这类 -olic 结尾的词, 就填了 olic, 整个是 vibolic 完全不是单词. 应该是 vibrant,
词根 vibr- 是震动, 引申活泼鲜明. 常见搭配 vibrant atmosphere / city / culture.

翻译:
文艺复兴是欧洲14到17世纪的文化复兴时期, 文化、艺术和智识上全面复苏, 强调人文主义
和个人成就.

生词:
Renaissance 文艺复兴
vibrant 充满活力的
intellectual 智识的
humanism 人本主义
```

**关键**：
- 题号是 `1）` 不是 `Q1` `1.`
- 错因先讲自己怎么填的（暴露思路）
- 然后给正确填法 + 词根解释
- 生词是 1 行 1 个 `word 中文`, 不要三栏字典
- 不写完整答案列表
- 翻译简短意译, 不用逐句雕琢
- 不抄题目


## 学术阅读的对比

### 手写版

```
学术阅读
M1
以下哪一项最有可能是关于太阳"此前被广泛接受的结论"？

关键词：previously accepted（过去认为的、旧有共识）→ 说明现在的新发现可能推翻或挑战它。

新观测发现：在Perseus分子云中，最年轻的恒星全都是成对出现的（binary twins）。
而较老的双星系统，两颗星距离更近。
这些发现对"关于太阳的旧有结论"有"令人惊讶的暗示"。
→ 也就是说，太阳可能也属于某种"双星系统"或"单星系统"，而新发现暗示"恒星普遍成对形成"，
这可能挑战了"太阳是单星"的旧观点。因此选A。
```

**特征**：
- 整道题一气呵成，不严格分段
- 关键词 → 新发现 → 推断 → 选项，一步步推
- 没"框架/主旨/细节"那种分类
- 选项只写对的


### 目标版（agent 输出）

学术阅读题如果学生是单纯推断错的，可以参考手写版"一气呵成"风格，不必硬套"框架/主旨/细节"。

```
M1
学术阅读
(材料: 太阳与双星系统, 4 段)

13）题: 关于太阳"此前被广泛接受的结论"最有可能是哪个.
我选 B (The Sun is part of a binary system).

错因: 我把"surprising implications for..."理解成"新发现说明太阳属于双星".
但题干锁定的就是"previously accepted"旧观点, 原文既然"新发现"是恒星普遍成对,
那"旧观点"必然是反过来的"太阳是单星". 正确答案 A 就是这个.

主旨: 新观测发现年轻恒星几乎全是双星, 挑战了"太阳是单星"的旧共识.
```

**说明**：
- 用了中文括号标材料主题（手写版也常用）
- 错因口语化 + 自己说一遍题
- 没有"框架/主旨/细节"硬分级
- 没用框线
- 只写错的那个 + 正确那个一句, 其它不提


## 小结：什么时候该用什么模板

| 题型 | 处理方式 |
|---|---|
| 词汇填空（错 1-2 个） | 错因 + 简短翻译 + 生词一行一个 |
| 词汇填空（错 ≥3） | 短模板，只列错词 + 释义 |
| 生活类阅读 | 题 + 错因 + 只分析错的选项 + 一句话主旨 |
| 学术阅读（短） | 题 + 错因 + 一气呵成推理 |
| 学术阅读（长） | 题 + 错因 + 只分析错的选项 + 轻量主旨 |
| 听并回应 | prompt + 错因 + 只写错的那个 |
| 对话 / 讲座 / 公告 | 完整 transcript（必要）+ 题 + 错因 + 只分析错的 |


## 附录：快进抓取代码段（直接复制用）

SKILL.md 的"快进模式"配套。**遇到错题分析任务时直接套下面的 eval**，不重新发明：

### 找 homework tab + 进答题解析

```js
() => {
  // 找作业详情 tab（href 含 /homework/student/）
  const a = document.querySelector('a[href*="/homework/student/"]');
  if (a) a.click();
  return 'clicked: ' + (a ? a.href : 'not found');
}
```

### 切 section 按钮

```js
() => {
  const b = Array.from(document.querySelectorAll('button')).find(b => b.textContent.trim() === '阅读M1');
  if (b) b.click();
  return 'clicked';
}
```

### 拿当前 section 错题列表

```js
() => {
  const items = Array.from(document.querySelectorAll('div'))
    .filter(d => /^\(\d+\)\s*[✓✗]$/.test(d.textContent.trim().replace(/\s+/g, '')))
    .filter(d => !d.querySelector('div'));
  const seen = new Set();
  return items.map(d => {
    const m = d.textContent.match(/\((\d+)\)\s*([✓✗])/);
    return m ? {n: +m[1], wrong: m[2] === '✗'} : null;
  }).filter(Boolean).filter(i => seen.has(i.n) ? false : (seen.add(i.n), true));
}
```

### 一次性 click 所有错题 + 收集右栏 innerText

```js
(wrongNums) => new Promise(async resolve => {
  const results = {};
  const allItems = Array.from(document.querySelectorAll('div'))
    .filter(d => /^\(\d+\)\s*[✓✗]$/.test(d.textContent.trim().replace(/\s+/g, '')))
    .filter(d => !d.querySelector('div'));
  for (const n of wrongNums) {
    const item = allItems.find(d => d.textContent.includes('(' + n + ')'));
    if (!item) { results[n] = 'NOT FOUND'; continue; }
    item.click();
    await new Promise(r => setTimeout(r, 400));
    const detail = document.body.innerText;
    const start = detail.search(/(题目前引导|题干|得分)/);
    let end = detail.length;
    const nextMatch = detail.slice(start + 50).match(/\n\(\d+\)\s*[✓✗]/);
    if (nextMatch) end = start + 50 + nextMatch.index;
    results[n] = detail.slice(start, end);
  }
  resolve(results);
})
```

### 一次性触发所有"原文" popover 拿 transcript

```js
(wrongNums) => new Promise(async resolve => {
  const tr = {};
  const allItems = Array.from(document.querySelectorAll('div'))
    .filter(d => /^\(\d+\)\s*[✓✗]$/.test(d.textContent.trim().replace(/\s+/g, '')))
    .filter(d => !d.querySelector('div'));
  for (const n of wrongNums) {
    const item = allItems.find(d => d.textContent.includes('(' + n + ')'));
    item.click();
    await new Promise(r => setTimeout(r, 400));
    const btn = Array.from(document.querySelectorAll('button, div, span'))
      .find(e => e.textContent.trim() === '原文' && e.children.length === 0);
    if (btn) {
      btn.scrollIntoView({block: 'center'});
      btn.click();
      await new Promise(r => setTimeout(r, 300));
      const pop = Array.from(document.querySelectorAll('.el-popover'))
        .find(p => getComputedStyle(p).display !== 'none' && p.offsetHeight > 0);
      tr[n] = pop ? pop.innerText : '(no popover)';
    } else {
      tr[n] = '(no 原文 btn)';
    }
  }
  resolve(tr);
})
```

**注意**：playwright-cli 调用时把 `wrongNums` 序列化成 JSON 字符串传过去：

```bash
playwright-cli --s=default eval "<上面的箭头函数代码>" '[5,14,26,27]'
```
