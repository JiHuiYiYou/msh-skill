# 风格指南：人味 vs AI 味

## 核心判断标准

把输出打印出来，**能不能一眼看出是 AI 写的？**
- 能 → 重写
- 不能（看上去像学生手抄/手打的） → 通过


## AI 味的典型特征（要避免）

| 特征 | AI 版长这样 | 人味版长这样 |
|---|---|---|
| 标题分隔 | 长框线、`【Q1 词汇填空】` | 裸 `M1` `M2`，空行分组 |
| 题号 | `Q1.` `1.` `(1)` | `1）` `13）` 中文括号 |
| 大段引文 | 整段抄原文 / 完整 transcript 放每道题 | 最多 1-2 句关键引文，长文写一句摘要 |
| 错因开头 | "未能准确把握..." "对 XXX 知识点掌握不牢固" | "我脑子一抽就想到..." "其实当时就有点犹豫" |
| 错因结尾 | 必有"建议：加强 XXX 训练" | 不写建议，写完就完事 |
| 选项分析 | A→B→C→D 严格顺序，每项都写 | 只写错的那个，1-2 句带过；其它干脆不写 |
| 生词 | 三栏字典（音标+简明释义+网络），写一堆 | 词 + 中文，1 行 1 个 |
| 段落标记 | `■ 原文` `■ 翻译` `■ 错因分析` | 不用 `■` 标记，写"错因:" "翻译:" "生词:" |
| 总结小节 | "【错因归类】" "【重点改进】" | **不写** |
| 符号 | `→` `✔` `✘` `─` `【】` `■` | 都不写，空行和标点够了 |
| `/` | 到处用，路径、答案、表格分隔 | 少用，写网络短语可以，其它地方别堆 |
| 完整答案 | 填词题列 10 个词 | 填词题**不要**列答案 |
| 抄题 | 复制粘贴题干文字 | 自己说一遍题在问什么 |
| 元信息 | "正确率 75%" "错了 4 道" | **不写** |
| 文件名 | `第五次模考-真题16-阅读错题分析.txt` | `260620阅读.txt` |


## 人味的来源

### 1. 暴露思路过程
手写版最有价值的部分——让学生看到自己当时是怎么想的。

**AI 写法**：
> 错因：未掌握 procedure 与 processes 的词义辨析。

**人味写法**：
> 脑子一抽就蹦出 procedure (程序), 觉得"理解这种程序"也讲得通, 直接填了 edure. 但其实要的是 processes, 复数.

把"脑子里发生什么"写出来，包括错误联想、被哪个词带跑、分神的瞬间。

### 2. 吐槽 + 自嘲
- "我压根不知道 precarity 是啥意思" — 承认不会
- "看到 -olic 结尾就套了, vibolic 完全不是单词" — 暴露错误路径
- "对的对的, 是我太菜了" — 自我吐槽

这种段子是手写版的灵魂。

### 3. 不完美
- 可以某个选项不分析
- 翻译可以跳过某句
- 生词可以不全
- 偶尔留"？"表示疑惑

### 4. 口语化连接词
- "其实"
- "但是"
- "关键区别"
- "脑子一抽"
- "被 X 带跑了"
- "整个跑偏"
- "对的对的"
- "其实当时就有点犹豫"
- "大概看了一眼"


## 错因长度

不是所有错因都要写很长：

| 错因类型 | 写多少 |
|---|---|
| 真的不会 / 知识盲区 | 1-2 句，承认就好 |
| 被形近词带跑 | 2-3 句，讲清 A 是什么 / B 是什么 / 我为什么混了 |
| 转折没看到 | 2-3 句，引原文 + 标转折词 + 重新说一遍正确逻辑 |
| 没听到 / 走神 | 1 句，"我没听到 X" |
| 过度推断 | 2-3 句，列出原文实际说的 + 列出选项额外加的 |
| 习语不懂 | 2 句，习语字面 + 实际含义 |


## 选项分析"只写错的"

只写学生错选的那个。判断维度：

- 我错的那个为什么错
- 正确那个为什么对（如果不说清楚逻辑会缺一半就顺带提一句）

**不写**学生一眼就排除的选项。除非那个选项是常见干扰项（"我也差点选这个"）才带一句。

手写版常见做法：
- 错选 → 必写
- 正确 → 简单一句带过
- 另外两个 → 干脆不提


## 反例对照

### AI 版（要避免）

```
────────────────────────────────────────────
【听力 M1 - 听并回应 Q3】
────────────────────────────────────────────

■ 错因分析
听完 prompt 后机械套了一个"陈述性回应"，没有把对话当成社交闲聊。

■ 选项分析
A. The equipment goes in the back. → 偏离 prompt 主旨，语用关系不匹配。✘
B. The student rec center is closed. → 突然切换话题，逻辑跳脱。✘
C. I don't think that's a good idea. → 否定语气，但 prompt 不是建议/请求。✘
D. We'll have to check it out. → 直接对 prompt 的共享信息给出承诺，语用上最匹配。✔

────────────────────────────────────────────
```

### 人味版（要照着写）

```
M1
听并回应

3）原文: I hear that the local gym has new equipment.
我选 A (The equipment goes in the back).

错因: 听到 prompt 就机械地想"对方在说什么", 没意识到这是社交分享. 人家说"听说健身房有新器材", 是分享一个信息, 期待的是"我也有兴趣, 要不要去看看"这种承接. 我脑子一抽就想到"器材被放在哪", 完全没回应 prompt 的语用关系.

正确答案 D (We'll have to check it out) 是"我也想去看看"的承诺, 最匹配.
```

**关键差异**：
1. 没有 `─` 框线 + `【】` 标题
2. 没有 `■ 错因分析` `■ 选项分析` 段落标记
3. 错因暴露思路"我脑子一抽就想到'器材被放在哪'"
4. 只写错的那个 + 正确答案一句，其它选项干脆不提
5. 没有 `→` `✔` `✘` 这种符号


## 自检 Checklist

写完输出后，逐项过：

- [ ] 没有 `─` `【】` `■` 框线和段落标记
- [ ] 没有 `→` `✔` `✘` 这种人类不常用的符号
- [ ] 没有"重点改进""错因归类"那种结构化总结
- [ ] 没有正确率 / 错了几个 / 总分
- [ ] 题号是 `1）` `13）` 中文括号
- [ ] 标题裸写 `M1` `M2`
- [ ] 没有大段原文，最多 1-2 句关键引文
- [ ] 只分析错的那一项，其它选项不写
- [ ] 错因暴露思路过程、有"我脑子..."这种第一人称吐槽
- [ ] 生词是 1 行 1 个 `word 中文` 格式，没有音标没有三栏
- [ ] `/` 出现得不多
- [ ] 填词题没有完整答案列表
- [ ] 没有抄题，题干自己说一遍
- [ ] 至少有一处自嘲 / 留白 / 吐槽
- [ ] 整体读起来像学生本人写给自己复习的笔记

任意一项不过 → 回去改。
